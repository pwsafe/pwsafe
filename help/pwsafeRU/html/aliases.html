Псевдонимы для паролей

Если вы используете системы «единого входа», которые позволяют использовать один и тот же пароль для различных машин/серверов/приложений, то можно настроить несколько элементов Password Safe таким образом, чтобы изменение пароля в одном из них автоматически отражалось в остальных. Связывание элементов таким образом производится с помощью псевдонимов. Этот раздел описывает порядок определения и использования псевдонимов.

Краткое описание

Основная идея состоит в том, чтобы пароль одного элемента мог быть ссылкой на пароль другого элемента. Элемент, на который ссылаются называется исходным, а ссылающиеся на него именуются псевдонимами. Если элемент является псевдонимом для другого, его пароль совпадает с исходным. Так, при копировании в буфер обмен пароля псевдонима, туда будет помещён пароль исходного элемента. Изменение пароля исходного элемента немедленно отражается на связанных с ним псевдонимах. На первый взгляд это может показаться сложным, но, на самом деле, всё достаточно просто и понятно. Самая сложная часть — это определение псевдонимов.

Определение псевдонимов

Если в Password Safe пароль элемента записать определённым образом, он будет ссылался на пароль другого элемента. Для этого в поле пароля псевдонима нужно указать «имя» исходного элемента в квадратных скобках. Чаще всего под «именем» можно понимать заголовок элемента. Таким образом, если заголовок исходного элемента «Главный», то указание «[Главный]» (без кавычек) вместо пароля другого элемента делает его псевдонимом для элемента «Главный».

В общем случае, «имя» может содержать значения полей «группа», «заголовок» и «имя пользователя», разделённые двоеточием. Заметьте, что если заголовок уникален в данном контейнере, то другие поля являются необязательными. Аналогично, если пара «группа» и «заголовок» или «заголовок» и «имя пользователя» однозначно определяет элемент, то значение третьего поля можно не указывать.

В элементах контейнера Password Safe, только поля «заголовок» и «пароль» являются обязательными для заполнения. Поля «группа» и «имя пользователя» необязательны если комбинация «группа/заголовок/имя пользователя» уникальна. Из описанного следует, что «пароль» для псевдонима имеет вид [g:t:u] (g – «группа», t – «заголовок», u – «имя пользователя»). При этом достаточно указать только ту информацию, по которой можно однозначно определить элемент. То есть, если в контейнере лишь один элемент с заголовком «t», то в качестве пароля псевдонима можно указать [t] (если указано только одно значение, то считается, что это заголовок, т. к. он является обязательным). В случае, если несколько элементов имеют одинаковый заголовок, будет выведено предупреждение, и вам потребуется указать дополнительную информацию об исходном элементе (группу и/или имя пользователя). Допускается любой из вариантов записи [g:t:u], [g:t], [t:u] или [t], если от указывает на единственный существует элемент. Указание двоеточия без значения равносильно пустому полю, например, [g:t:] указывает на элемент с заголовком «t» в группе «g» без имени пользователя, а [:t:] равносильно элементу «t» без имени пользователя, находящемуся в корне дерева.

При изменении пароля для псевдонима (вручную, или с помощью кнопки «создать пароль»), правки будут применены только после нажатия на кнопку «Применить» или «OK». Если же требуется изменить пароль исходного элемента псевдонима, то для этого необходимо открыть и отредактировать сам исходный элемент. При изменении пароля псевдонима с [g:t:u] на какое-либо значение в другом формате, его тип изменится на «обычный», а связь с исходным элементом будет разорвана. Чтобы вернуть разорванную связь можно использовать отмену последнего действия или заново задать пароль ссылку в поле пароля.

В отличие от ярлыков, которые наследуют значения для всех полей, псевдонимы могут иметь свои значения во всех полях кроме пароля, который наследуется от исходного элемента. Например, при копировании ссылки из псевдонима, будет использовано значение, указанное в псевдониме, но при копировании пароля, будет подставлено значение из исходного элемента.

Если для «обычного» элемента была включено история паролей, а затем он был преобразован в псевдоним (за счёт изменения пароля),то история не будет удалена. Если этот элемент будет преобразован в «обычный», история снова станет активной и будет обновляться при последующих сменах пароля.

Ниже приведены поясняющие примеры.

Примеры

  1. Если заголовок исходного элемента «Сервер» и никакой другой элемент не имеет такого заголовка, то элемент, вместо пароля которого указано «[Сервер]», будет являться его псевдонимом.
  2. Если элемент «Сервер» присутствует в двух группах, например «Отдел A» и «Отдел B», тогда первый определяется сочетанием «[Отдел A:Сервер]». Заметьте, что нет необходимости писать имя пользователя, если указанной информации достаточно для однозначного определения элемента.
  3. Имя пользователя тоже может использоваться для различения похожих исходных элементов, например «[Сервер:Вася]» и «[Сервер:Маша]».
  4. Наконец, можно указать все три поля, например, «[Отдел A:Сервер:Вася]».

Вид

В древовидном списке исходные элементы, на которые указывает хотя бы один псевдоним, помечены знаком зелёного треугольника вместо зелёного квадрата.

Псевдонимы помечены серыми треугольниками.
