Параметры командной строки

При запуске Password Safe без параметров выдаётся запрос пароля для последнего открытого контейнера. Если контейнеры на этом компьютере ещё не открывались (например, при первом запуске Password Safe), будет предложено создать новый контейнер. Это поведение можно изменить, вызвав Password Safe с параметрами:

  • pwsafe контейнер
    Будет открыт указанный в параметре файл вместо того, что использовался в прошлый раз. Если указано только имя файла (без пути), то поиск файла производится в каталоге, из которого было запущено приложение. Обратите внимание, что если имя файла содержит пробелы, то его нужно заключить в двойные кавычки ("имя файла").
  • pwsafe -r [контейнер]
    Указанный контейнер будет открыт в режиме «только чтение». Если контейнер не указан, будет запрошено имя контейнера, который затем будет открыт в режиме «только чтение».
  • pwsafe -e имя_файла
    Указанный файл будет зашифрован с использованием ключа, полученного из введённого пароля. Примечание. Файл может быть любым. Имя зашифрованного файла останется тем же, а в качестве расширения добавляется «.psf».
  • pwsafe -d имя_файла
    Указанный файл будет расшифрован с использованием ключа, полученного из введённого пароля. Примечание. Данный метод применим только к файлам ранее зашифрованным с использованием параметра «-e».
  • pwsafe -c
    В данном случае приложение будет запущено без контейнера, в том числе не будет выведен стандартный диалог (для открытия контейнера используйте меню «Файл»).
  • pwsafe -s [контейнер]
    Приложение будет запущено «молча» — в свёрнутом виде и без контейнера (если он не указан в параметрах). При разворачивании окна будет выведен диалог открытия контейнера (параметр предназначен для запуска Password Safe через ярлык в меню автозапуска). Примечание. в данном случае значок приложения будет помещён в область уведомлений (трей).
  • pwsafe -m
    Параметр аналогичен «-c», но, запуск производится в свёрнутом виде.

Данные параметры могут оказаться полезными, если вы желаете использовать одни и те же настройки на разных машинах, например, при запуске Password Safe со съёмного диска.

  • -u имя_пользователя
    При запуске, вместо настроек пользователя, вошедшего в систему, будут использоваться настройки, заданные для пользователя имя_пользователя.
  • -h имя_хоста
    При запуске, вместо настроек для текущей машины, Password Safe будет использовать настройки, указанные для имя_хоста.
  • -g файл_настроек
    В данном случае указанный файл будет использован для загрузки и сохранения настроек (вместо файла по умолчанию «pwsafe.cfg»). Если указано только имя файла (без пути), то поиск файла производится в каталоге, из которого было запущено приложение, но каталог может быть переопределён через переменную окружения PWS_PREFSDIR (см. описание в разделе Настройки). Обратите внимание, что если имя файла содержит пробелы, то его нужно заключить в двойные кавычки ("имя файла").

Примечание. Т. к. файл настроек использует формат XML, имя пользователя и имя ПК должны соответствовать требованиям стандарта:

  • Начинаться с буквы или цифры. Если это не выполняется, будет добавлена приставка: «u» к имени пользователя, «h» к имени ПК.
  • Другие символы должны быть цифрами, буквами, или одним из следующих символов: подчёркивание («_»), дефис («-»), двоеточие («:») или точка («.»). Любые другие символы будут заменены на подчёркивание.

Кроме того есть несколько параметров начинающихся с «--»:

  • --do-tall, --do-wide, --do-auto
    По умолчанию Password Safe проверяет разрешение экрана и выбирает, какой вариант («высокий» или «широкий») использовать для больших окон. --do-tall принудительно устанавливает «высокий» вариант, --do-wide — «широкий», а --do-auto сбрасывает предпочтения и возвращает автоматический выбор. Эти три параметра изменяют настройки приложения, поэтому, для установки предпочтений на данной машине достаточно запустить Password Safe с требуемым параметром один раз.
  • --novalidate
    Не использовать автоматическую проверку целостности при открытии контейнера. Есть несколько обязательных проверок, которые производятся независимо от этого параметра, например проверка уникальности идентификатора элемента и сочетаний Группа/Заголовок/Имя пользователя.
  • --testdump
    Используется для проверки механизма создания минидампов процесса Password Safe.
  • --cetreeview
    Новая функция: позволяет выделять и сравнивать два элемента через меню «Правка» или контекстное меню элемента. Выделение нескольких элементов поддерживается в простом списке и не поддерживается в древовидном. Данный ключ добавляет в древовидном режиме пункт меню «Сравнить элементы...» и позволяет выбрать второй элемент из отдельного списка. Функционал множественного выделения в древовидном режиме находится в разработке. После его реализации данный параметр будет игнорироваться.
