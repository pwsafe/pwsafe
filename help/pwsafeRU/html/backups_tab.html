Вкладка «Резервные копии»

Вкладка «Резервные копии» содержит параметры создания автоматических резервных копий.

Сохранять контейнер после вставки или редактирования

Если включено, сохранение происхоидт при любом изменении, будь то вставка, удаление, дублирование, редактирование или переименование элемента или группы, в том числе, если вставка или перемещение произошли с помощью импорта или перетягивания из другой группы/контейнера. При использовании данного параметра также рекомендуется включить «Создавать копию (.ibak) перед сохранением», чтобы предотвратить возможные повреждения, вызванные проблемами с диском.
Примечание. Изменения, производимые через диалог результатов сравнения сохраняются не сразу, а после закрытия диалога сравнения.

Создавать копию (.ibak) перед сохранением

Если помечено, Password Safe будет копировать контейнер перед каждым сохранением. Имя файла, его расположение и количество сохраняемых копий указаны в следующих полях.

Имя для резервной копии

Имя для резервной копии состоит из начала (по умолчанию это имя файла контейнера без расширения «.psafe3»), и суффикса (может быть пустыми или содержать номер/дату/время резервного копирования). Здесь можно настроить именование резервных копий и их количество. Password Safe автоматически перезапишет старую копию при превышении указанного количества, если для создания суффикса используется «Автонумератор».

Каталог для резервных копий

По умолчанию Password Safe создаёт резервные копии в том же каталоге, где находится открытый в настоящий момент контейнер. В этом поле можно указать другое расположение, например, отдельный каталог для резервных копий, сетевой диск, съёмный накопитель и т. п. Для корректной работы необходимо иметь права на запись в указанный каталог.

При указании каталога для резервных копий допускается использование переменных окружения, например «%homedrive%%homepath%\Documents\PWS Backups\». Раскрытый путь к каталогу будет указан под полем ввода при переключении на другое поле или вкладку.
