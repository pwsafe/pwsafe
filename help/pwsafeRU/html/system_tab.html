Вкладка «Система»

Вкладка «Система» позволяет задать настройки, связанные с взаимодействием Password Safe и ОС.

Отображать значок в области уведомлений

Если помечено, Password Safe будет сворачиваться в значок в области уведомлений (трее). Если флажок снят, при сворачивании Password Safe будет оставаться панели задач.

Значок в области уведомлений показывает, является ли контейнер заблокированным () или открытым (). Если контейнер заблокирован, вас попросят ввести мастер-пароль при разворачивании окна приложения.

Хранить N использованных элементов в меню значка

Если установлен флажок Отображать значок в области уведомлений, вы можете щёлкнуть правой кнопкой мыши на значке и выбрать один из недавно использованных элементов без отображения главного окна программы. Данная опция позволяет задать количество элементов, сохраняемых в этом списке. Примечание. Список сохраняется только при сохранении или закрытии контейнера, открытого в режиме «чтение/запись».

Скрывать значок (должна быть задана горячая клавиша)

Если помечено, Password Safe будет скрывать значок в области уведомлений. Данная функция доступна только в том случае, если задана комбинация клавиш, позволяющая отобразить окно Password Safe.

Запускать Password Safe при входе в систему

Если помечено, при входе в систему Password Safe будет автоматически запускаться и помещаться в область уведомлений. Примечание. Мастер-пароль будет запрошен только тогда, когда вы дважды щёлкнете по значку левой кнопкой.

Недавно использованные контейнеры

Здесь вы можете указать количество запоминаемых контейнеров, отображаемых в меню «Файл». Примечание. Для вступления изменений в силу необходимо перезапустить Password Safe.

Отображать недавно использованные контейнеры в меню Файл, а не в подменю

Данная настройка управляет расположением списка недавно использованных контейнеров (внутри самого меню «Файл» или в подменю).

По умолчанию открывать контейнер только для чтения

Если помечено, Password Safe будет по умолчанию отрывать контейнеры в режиме «только чтение». Данная настройка может пригодиться, если контейнером пользуются несколько людей, так как по умолчанию Password Safe разрешает запись первому, открывшему контейнер, и блокирует изменения для остальных. С её помощью вы сможете сами выбрать, кто должен иметь права на запись.

Разрешить одновременный запуск нескольких копий (для применения требуется перезапуск всех копий программы)

Убрать настройки Password Safe из реестра Windows

До версии 3.05, Password Safe использовал реестр Windows для хранения некоторых настроек (список ранее открытых контейнеров, изменённые настройки, и пр.). Начиная с версии 3.05 настройки были разбиты на две группы: связанных с контейнером (хранятся в контейнере) и связанные с пользователем/машиной (хранятся в отдельном файле настроек). Настройки будут удалены из реестра после установки флажка и нажатия кнопки Применить. Данный функционал может быть использован для удаления следов Password Safe с вашего компьютера. Но, в случае, если вы будете использовать более старые версии, некоторые настройки Password Safe примут своё изначальное значение.

Примечания:

  • Если предыдущие версии Password Safe не использовались на данном компьютере, или, если данные из реестре были удалены ранее, флажок и кнопка Применить будут скрыты.
  • Реестр будет использован лишь в тех редких случаях, когда Password Safe не сможет создать или произвести запись в файл с настройками.

Переместить файл настроек в каталог пользователя

До версии 3.22, Password Safe сохранял изменения настроек (пользовательские настройки, список ранее открытых файлов, размеры окон и пр.) в том же каталоге, где находился исполняемый файл (pwsafe.exe). В новых версиях Windows это вызвало ряд проблем, связанных с тем что пользователи не имеют прав на запись в этот каталог. Поэтому, начиная с версии 3.22, файл настроек (pwsafe.cfg) создаётся в стандартном для файлов такого рода месте — в профиле пользователя внутри каталога «Application Data».

При обновлении версии существующий файл настроек не переносится автоматически («если что-то работает, не трогай это»). Но, если вы хотите перенести настройки, нужно поставить флажок и нажать кнопку «Применить».

Файл с настройками поддерживает хранение настроек нескольких пользователей, возможно, работающих с разных машин. Данная функция производит перенос настроек текущего пользователя (связанных с данным компьютером) в файл по новому пути. Если присутствуют настройки для других пользователей/компьютеров, они останутся в исходном файле. Если же после переноса настроек файл окажется пустым, он будет удалён.

Если Вы желаете использовать общий файл настроек для нескольких пользователей/компьютеров, оставьте его в том же каталоге, где находится исполняемый файл, или укажите путь к нему при запуске (через ключ «-g файл_настроек», описанный в справке по параметрам командной строки).
