Pestaña sistema

La pestaña sistemas da acceso a opciones relacionadas con la interacción de Password Safe con el sistema en que se ejecuta.

Mostrar icono en el área de notificación

Si está seleccionado, Password Safe se minimizará al aréa de notificación del sistema. Si no está seleccionado, el programa se minimizará a la barra de tareas.

El icono en el área de notificación indica si la base de datos está bloqueada () o desbloqueada (). Si está bloqueada, se solicitará la contraseña maestre al volverla a primer plano.

Recordar últimas N entradas usadas en el menú del área de notificación

Si la opción Mostrar icono en el área de notificación está activada, puedes hacer clic derecho en el icono del área de notificación para obtener las entradas usadas más recientemente sin ir a la pantalla de la aplicación. Esta opción te permite configurar cuantas entradas recientes quieres mantener en las lista del icono del área de notificación.

Ocultar icono (la tecla de acceso DEBE estar activada)

Si está seleccionada, hará que Password Safe ocurte su icono en el área de notificación. Esta funcionalidad sólo está disponible si la tecla rápida para iniciar o reabrir Password Safe está activada.

Iniciar PasswordSafe al iniciar sesión

Si está seleccionada, hará que se inicie Password Safe automáticamente en el área de notificación al iniciar sesión. Se te pedirá la contraseñas maestra la primera vez que hagas doble clic en el icono del área de notificación, no antes.

Bases de datos Passwordafe recientes

Este control te permite configurar cuantas bases de datos abiertas recientemente se muestran en el menú Archivo. Debes salir y reiniciar Password Safe para que tengan efecto los cambios en este control.

Bases de datos recientes en el menú archivo en vez de en un submenú

Esta opción determina como se presentan las bases de datos abiertas recientemente en el menú archivo - directamente en el menú o como un submenú.

Abrir base de datos en sólo lectura por defecto

Fijar esta opción hará que Password Safe abrás las bases de datos en modo sólo lectura por defecto. Esto puede ser util en un entorno multiusuario, como por defecto Password Safe permite acceso de escritura a la primera persona que abre la base de datos, restringe a las demás a modo sólo lectura. Con esta opción, puedes determinar por la política del sitio quién tendra acceso de escritura a una base de datos dada.

Permitir multiples instancias (todas las instancias deben reiniciarse)

Eliminar configuración de PasswordSafe del registro

Con anterioridad a la versión 3.05, Password Safe utilizaba el registro para almacenar datos de configuración, tales como configuración de opciones, última base de datos abierta, etc. A partir de esa versión, esa información de configuración está partida entre la específica de la base de datos y la específica del usuario/máquina, esta última está almacenada en un archivo de configuración. Seleccionar esta opción y pulsar el botón Aplicar hace que se borren las opciones en el registro. Esto es útil si quieres eliminar todo rastro de Password Safe de tu PC. De cualquier forma, las antiguas versiones de Password Safe volverán a sus valores por defecto para algunas opciones de configuración.
Notas:
  • Si nunca se ha instalado una versión previa de Password Safe, o si los datos ya han sido eliminados, la casilla de configuración y el botón Aplicar no se mostrarán.
  • En algunas extrañas circunstancias, en que Password Safe no puede crear o escribir en el fichero de configuración, se utilizará el registro.

Migrar configuración de PasswordSafe al directorio del usuario

Con anterioridad a la versión 3.22, PasswordSafe mantenía las preferencias de la aplicación (como algunas opciones de usuario, últimos archivos abiertos, tamaño y posición de la ventana, etc.) en un archivo en el mismo directorio del ejecutable de PasswordSafe. Esto ha causado problemas con versiones recientes de Windows en que los usuarios no tienen acceso de escritura en este directorio. Por tanto, a partir de la versión 3.22, el archivo de configuración (pwsafe.cfg) se creará en la ubicación estandar para tales ficheros (en el directorio de usuario 'Configuración local\Datos de programa').
Por defecto, las actualizaciones de instalaciones no cambian el archivo de preferencias existente ("si no está roto, no lo arregles"). De cualquiere modo, si un usuario quiere cambiar a la nueva configuración manualmente, seleccionando esta opción y pulsando en 'Aplicar' se llevará a efecto.

Mas detalladamente: El archivo de configuración está diseñado para soportar múltiples usuarios en múltiples máquinas. La función de migración cambiará la configuración del usuario actual (asociado con el actual PC) del archivo de configuración a una nueva ubicación. Si hay otras configuraciones propias de otro usuario/máquina en el archivo no se tocarán. Si no hay otras configuraciones tras la migración, se eliminará el archivo.
Si quieres compartir un archivo de configuración común entre usuarios y/o entre distintos PCs, puedes mantener el archivo en directorio del ejecutable, o especificar su localización explicitamente utilizando la opción '-g archivo-configuración', tal como se describe en argumentos de la línea de comandos.
