快捷键/加速键

以下是作为 PasswordSafe 相关的自定义快捷键性能限制详细资料。

这些组合键已经过102-键扩展的英国键盘上的测试。标准字符与拉丁字符应该都安装在任何其它键盘上。 一些组合键可能在这些键盘上存在差异，如：在英国键盘上“#”符号有它自己的键（具有加在西班牙语n字上的发音符号~）； 但在美国键盘上则是 3 字符（即 Shift+3）。 请仔细阅读此表并变换适合你的特殊键盘-你总能看到具体的实验某一组合键是如何工作的。

依你的键盘变换，某些键和 Alt 键已被保留。相对的 Alt 常被用于调用主菜单。按 Alt 键主菜单字母将出现下划线。 对于英文版软件，Alt 键总是存在，如： Alt+F (File), Alt+E (Edit), Alt+V (View), Alt+M (Manage) 以及 Alt+H (Help)。 对于非英文版软件，请参考这些下划线字母在你的主菜单上将如何为你预定的。 例如：在德文版 "File" 菜单变成 "Datei" 因此 Alt+D 将预留取代英文版通常的 Alt+F； 中文版则中文菜单名称后加括号的英文大写字母，该字母完全继承于英文版相同字母。如：文件(F)、编辑(E)等。

Enter（回车键） 也是作为预留给快捷方式的，其值被预设到 管理->选项->快捷方式属性页。因此，不能分配给该键其它功能。 请注意“展开/收缩组”功能在树目录视图模式是通过选择组和按住回车键实现的。 虽然，该键可能已被改变到其它任何有效的快捷方式， 作为使用中的回车键功能，在树目录视图状态下其原来功能仍然有效。 基于此原因，该功能不能通过快捷方式属性页改变。

Escape（换码键） 是预留作为关闭查找工具栏或最小化 Password Safe 到系统托盘的快捷键。 它也相当于在任何对话框上的取消按钮。

按照设计，快捷方式值作为在只读模式查看项目与在读写模式状态下编辑项目是一样的。它们不能单独设置。 同样，“删除项目”、“删除组”和“重命名项目”以及“重命名组”也都一样。 这些显示在快捷方式属性页上的内容仅作为各自“删除”和“重命名”之用。

部分键是预留给 Windows 系统的-如：Ctrl+Alt+Delete; Tab; Shift+Tab; Alt+Tab;Alt+Shift+Tab 等。

非标准键不支持（如：只有在汉字键盘上独有的键）。

PasswordSafe 快捷方式设置不承认不同之间的 Alt, Ctrl 和 Shift 键，如已设置为键盘左边的键时，右边相同的键则不被承认有效。 这也许会让一些组合键在某些键盘上无法使用。例如：欧洲符号在英国 102 扩展键盘右 Alt 键(Alt Gr)和 4 键。 不管怎样，按该组合键都将导致 Ctrl+Alt+4 的快捷方式结果。 因为，这是微软系统热键进程引起，而不是 Password Safe 所限制的。

备注: 设置非标准键可能使这些特定的快捷方式无法使用。如果你在使用其它电脑由于键盘没有这些键， 建议您只从下列这些表中选择组合键。
