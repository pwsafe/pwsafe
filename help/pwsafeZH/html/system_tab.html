系统选项卡

系统选项卡提供访问设置 Password Safe 涉及系统在运行时的相互作用。

在系统托盘放置图标

如勾选， Password Safe 将最小化至系统托盘区域；如不勾选，程序将最小化至任务栏。

系统托盘图标将指示密码数据库是否被锁定。锁定为 ()，未锁定为 ()。如数据库被锁定，当用户从最小化恢复时将被提示输入主密码。

记住最后 N 个在系统托盘菜单使用过的项目

如放置图标在系统托盘选项被启用，可以在没有经过程序主窗口时，在系统托盘图标右击直接转到大部分最近用过的项目。 该选项允许用户配置多少最近使用过的项目保存在系统托盘图标列表中。

启动系统时自动运行 Password Safe

如勾选该项，当启动系统时将导致 Password Safe 自动启动在系统托盘。 注意，在首次双击系统托盘图标时程序将提示输入保险箱密码。

最近 Passwordsafe 数据库

该控制命令允许用户配置在文件菜单显示多少最近打开过的数据库。 注意需要退出和重启 Password Safe 以使改变生效。

最近数据库在文件菜单上作为子菜单使用

该选项确定如何让最近数据库呈现在文件菜单上 - 直接在菜单上或作为一个子菜单。

默认打开数据库为只读

设置该选项将导致 Password Safe 打开数据库为默认只读模式。 该选项也许有助于多用户环境，设为默认后的 Password Safe 允许由谁为第一人称写访问打开数据库与限制其它人在只读模式。使用该选项，用户可以通过位置策略限定谁可以写访问一个特定的数据库。

从注册表删除 Password Safe 数据

在 3.05 版本之前 Password Safe 用过注册表存储配置数据，例如: 非默认选项设置、最后打开的数据库等等。 从这个版本开始，该信息被分成数据库-特定的用户/计算机 特定的设置，后者存储在一个单独的配置文件中。 选择该选项和单击应用按钮将导致注册表设置被清除。这是有助于比如希望删除 Password Safe 在用户的 PC 上的遗迹。注意，不管如何， 该方法在旧版本的 Password Safe 上将恢复默认值给部份设置。
备注:
  • 如果早期版本的 Password Safe 从未被安装或假如数据已被删除，则勾选框和应用 按钮不被显示。
  • 在极少情况下，Password Safe 位置不能创建或写入配置文件时，注册表将被使用。
