Argumentos de la línea de comandos

Ejecutar Password Safe sin argumentos hace que se pregunte por la contraseña maestra de la última base de datos que se abrió, o por la contraseña maestra de una nueva base de datos si nunca fue abierta una en tu máquina (por ejemplo, la primera vez que utilizas Password Safe). Es posible modificar esto ejecutando Password Safe como sigue:

  • pwsafe basededatos
    Esto abrirá el archivo de base de datos especificado en lugar del último abierto. Si se indica un nombre de archivo, sin trayectoria, se buscará en el directorio en que se llama a la aplicación. Si el nombre de archivo y/o trayectoria contienen espacios deben encerrarse entre dobles comillas.
  • pwsafe -r [basededatos]
    Esto abrirá la base de datos especificada en modo sólo lectura. Si no se indica base de datos, la aplicación preguntará por ella, y será abierta en modo sólo lectura.
  • pwsafe -v basededatos
     Esto validará la base de datos tras la introducción de la contraseñas maestra, corrigiendo las inconsistencias que sean necesarias. Bastante útil cuando se trabaja con bases de datos escritas por otras aplicaciones.
  • pwsafe -e nombrearchivo
    Preguntará por una contraseña y encriptará el fichero con una clave derivada de esa contraseña. Nota: El archivo puede ser cualquiera. El archivo encriptado tendrá el nombre del archivo original, con ".PSF" añadodo a el.
  • pwsafe -d nombrearchivo
    Preguntará por una contraseña y desencriptará el archivo con una clave derivada des esa contraseña. Nota: Esto sólo funciona en archivos que fueron encriptados ejecutando pwsafe con la opción '-e' (ver arriba).
  • pwsafe -c
    Esto iniciará la aplicación cerrada, esto es, sin ninguna base de datos y sin el diálodo inicial de apertura (Para acceder a una base de datos usa el menú Archivo).
  • pwsafe -s [basededatos]
     Iniciará la aplicación "silenciosamente", esto es, minimizada y sin base de datos (salvo que se especifique una). Cuando la aplicación se pasa a primer plano, se presenta el dialodo de apertura (Esta opción es útil para iniciar la aplicación al iniciar sesión, a través de un atajo en la carpeta Inicio del usuario). Nota: Esto pone la apliación implicitamente en la bandeja de notificación.
  • pwsafe -m
    Esta es la misma opción que '-c', con el añadido de que la aplicación se inicia minimizada.
Adicionalmente, se aceptan las siguientes opciones:
  • -u usuario
    Hace que la aplicación lea y escriba las preferencias del usuario espeficado, en lugar de usar el del usuario de la sesión.
  • -h hostname
    Hace que la aplicación lea y escriba las preferencias del equipo, en lugar del nombre de la máquina.
  • -g archivo_configuración
    Hace que se use el archivo especificado para leer y guardar las preferencias, en lugar del archivo por defecto pwsafe.cfg. Si sólo se indica en nombre del archivo, sin trayectoria, se buscará en el directorio en que se llama a la aplicación. Si el nombre de archivo y/o trayectoria contienen espacios deben encerrarse entre dobles comillas.
Estos puede ser útiles si quieres compartir las mismas preferencias en distintas máquinas, por ejemplo, al ejecutar Password Safe desde una llave portátil.
